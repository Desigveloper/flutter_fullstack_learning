# Project 4 — Business Booking App

Reusable systems, real scheduling logic, production-level UI patterns.

What it is: A booking platform (think barbershop, clinic, or gym). Businesses list services, customers book time slots, owners manage calendars.
What you'll learn:

Reusable component architecture
Date/time handling and calendar logic
Availability and conflict detection
Riverpod for scalable state management
Custom UI components and design systems
Deep linking and notification groundwork

Why fourth: The Booking App is where you stop writing features and start writing systems. Every pattern here carries directly into Tier 3.
