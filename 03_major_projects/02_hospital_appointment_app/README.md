# Project 2 — Hospital Appointment App

Your first app with real-world user flows, authentication, and a backend.

What it is: Patients book appointments, doctors view their schedule, admins manage the system.
What you'll learn:

Firebase Authentication (register, login, logout)
Firestore as a database (real CRUD against a live backend)
Role-based UI (patient sees different screens from doctor)
Form handling at a production level
Connecting UI state to live data with StreamBuilder

Why second: You now understand Flutter and CRUD. This project adds auth and a real backend — the two things that turn a demo into an actual app.
