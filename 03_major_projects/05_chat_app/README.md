# Project 5 — Chat / Messaging App

Real-time data, streams, complex UI state — this is where most developers struggle.

What it is: A full messaging app. One-to-one chat, message status (sent, delivered, read), user presence (online/offline), media sharing.
What you'll learn:

Firestore real-time streams in a complex context
Optimistic UI updates (message appears instantly, then confirms)
Pagination (loading messages in batches)
Firebase Storage (image/file uploads)
Push notifications with Firebase Cloud Messaging
Advanced Riverpod patterns

Why fifth: Real-time data is the hardest concept in mobile development.
By this point you have the architecture knowledge to handle it without getting lost.
